# Digital Assets Repository

Central repository for storing and organizing digital assets for future projects.

## Folder Structure

```
digital-assets/
├── brand/
│   ├── logos/          # Logo files (PNG, SVG, etc.)
│   └── colors/         # Color palettes, swatches
├── social-media/
│   ├── templates/      # Reusable social media templates
│   └── graphics/       # Social media graphics and images
├── project-assets/
│   ├── images/         # General project images
│   └── documents/      # PDFs, documents, etc.
└── README.md
```

## How to Use

### Adding Assets
1. Place files in the appropriate folder based on type
2. Use descriptive file names (e.g., `edmund-bogen-logo-white.png`)
3. Commit and push changes

### Accessing Assets
- **Direct URL**: `https://raw.githubusercontent.com/edmundbogen/digital-assets/main/[path-to-file]`
- **Clone locally**: `git clone https://github.com/edmundbogen/digital-assets.git`

### File Naming Convention
- Use lowercase with hyphens: `my-image-name.png`
- Include descriptive context: `brand-logo-dark-background.svg`
- Add dimensions if relevant: `social-post-1080x1080.png`

## Supported File Types
- Images: PNG, JPG, SVG, GIF, WebP
- Documents: PDF
- Graphics: AI, PSD, Figma exports
